## 프로그램 개요

### 전체 베타 테스트 기간
- **총 4주 (1달)**
  - Week 1-3: 신규 모집 기간 (선착순 100명)
  - Week 4: 기존 참가자 프로그램 완료 기간

### 개인별 프로그램 기간
- **가입일로부터 7일**
  - Day 1: 집중 온보딩 (필수)
  - Day 2-7: 자유 사용 (선택)

### 목표
- 베타 테스터 **100명** 모집 (선착순)
- Day 1 완료율 **85%** 달성
- NPS 점수 **50+** 달성
- 최종 피드백 **55명** 이상 수집

### 핵심 철학
```
✅ 짧고 강렬함 (1주일)
✅ 명확한 마감 (Day 7)
✅ 즉각적 보상 (바로 지급)
✅ 부담 최소화 (알림 2번)
✅ 선택권 존중 (강제 X)
```

### 개인별 프로그램 (가입일로부터 7일)

#### Day 1: "25분 올인" (필수)

#### Stage 1: 첫 3분 체험 (5분)
```
미션: 가장 쉬운 도구 1개만 써보기
└─ QR 생성 또는 이미지 검색 (3분)

완료 즉시:
├─ 팝업: "🎉 첫 자동화 성공!"
├─ 크레딧 10개 즉시 지급
└─ 간단 설문 2문항 (30초)
    1. 쉬웠나요? 😊/😐/☹️
    2. 결과물 OK? 👍/👎

🎁 보상: 크레딧 10개 (1만원 상당)
⏱️ 소요: 5분
```

#### Stage 2: 실전 투입 (20분)
```
미션: 실무 시나리오 1개 완수
└─ 보고서 / 블로그 / 대량 QR 중 선택

타이머 자동 측정:
"평소: [수동 입력]분 걸렸어요"
"오늘: [자동 측정]분 걸렸어요"
→ 화면: "🎊 [XX]분 절약!"

간단 피드백 (5문항, 3분):
1. 시간 절약 체감?
2. 결과물 품질? (1-5)
3. 이 가격이면 구매? (1000/3000/5000/10000/안살듯)
4. 추천 의향? (NPS 0-10)
5. 불편한 점?

🎁 보상: 크레딧 20개 (2만원 상당)
⏱️ 소요: 20분
```

**Day 1 완료 시: 크레딧 30개 (3만원 상당) 획득**

---

### Day 2-6: "자유 탐험" (선택)

```
강제 미션 없음! 본인 원하는 대로 사용

옵션 1: 3회 이상 사용
└─ 도구를 3회 이상 사용하면 자동 지급
   → 크레딧 10개 (1만원 상당)

옵션 2: 친구 초대
└─ 1명 초대 시 (친구도 10개 받음)
   → 크레딧 20개 (2만원 상당)

옵션 3: 리뷰 작성
└─ 50자 이상 후기 작성
   → 크레딧 20개 (2만원 상당)
```

**알림: Day 3 (미사용자만), Day 6 (전체)**

---

### Day 7: "최종 피드백" (선택)

```
미션: 최종 설문 완료 (3분)

질문:
1️⃣ 이번 주 사용 횟수 (자동 계산)
2️⃣ 가장 유용했던 도구 TOP 3
3️⃣ 정식 런칭하면 구매 의향?
   └→ 구매 의향 있으면: 월 지불 가능 금액?
4️⃣ 최종 NPS (0-10)
5️⃣ 한 줄 평가 (선택)

🎁 보상: 크레딧 10개 (1만원 상당)
⏱️ 소요: 3분
```

## 구현된 기능

### 1. 타입 정의

**파일:** `src/types/beta-onboarding.ts`

```typescript
// 주요 타입
- OnboardingMission: 온보딩 미션 정의
- OptionalMission: 선택 미션 정의
- BetaTesterProgress: 전체 진행 상황
- Stage1Feedback, Stage2Feedback, FinalFeedback

// 헬퍼 함수
- formatCredits(amount): 크레딧 표시 포맷
- estimateUsage(credits): 사용 가능 도구 계산
- getBetaStatus(currentParticipants): 베타 상태 계산

// 베타 설정
- BETA_CONFIG: 기간, 인원, 날짜 설정
  - USER_PROGRAM_DAYS: 7일
  - TOTAL_BETA_WEEKS: 4주
  - RECRUITMENT_WEEKS: 3주
  - MAX_PARTICIPANTS: 100명
```

```typescript
interface BetaStatus {

  currentParticipants: number;  // 현재 참가자 수
  spotsRemaining: number;       // 남은 자리
  weekNumber: number;           // 현재 몇 주차 (1-4)
}

// 사용 예시
const betaStatus = getBetaStatus(currentParticipants);
if (betaStatus.isOpen) {
  // 신규 모집 중
} else if (betaStatus.spotsRemaining === 0) {
  // 정원 마감
} else {
  // 모집 기간 종료
}
```

## 사용법

### 1. 베타 신청

```
1. /beta 페이지 접속
2. 신청 폼 작성
3. "🎮 베타 미션 시작하기" 클릭
```

### 2. Day 1 온보딩

```
1. /beta/missions 접속
2. Stage 1: QR 또는 이미지 검색
   → 크레딧 10개 획득
3. Stage 2: 실무 시나리오
   → 크레딧 20개 획득
4. 설문 완료
   → Day 1 완료!
```

### 3. 선택 미션 (Day 2-7)

```
// 자유롭게 선택

- 리뷰 작성 → 크레딧 20개
```

```
1. /beta/feedback 접속
```

## 측정 지표

### ⭐ 최우선 지표 (Critical)

1. **완료율**: 각 Stage 완료 비율
   - Day 1 목표: **85%**
   - Day 7 목표: **55%**

2. **NPS 점수**: 50+ 목표
   - Stage 2에서 측정 (Day 1)
   - Day 7 최종 측정

3. **시간 절약 효과**: 평균 절약 시간
   - Stage 2에서 자동 측정

4. **유료 전환 의향**: 구매 의사 비율
   - "바로 구매" 비율
   - 지불 의향 금액

### 중요 지표 (Important)

5. 도구별 사용률 및 선호도
6. 가격 민감도 (지불 의향 금액)
7. 재사용률 (주 2회 이상 사용자)
8. 바이럴 계수 (친구 초대율)

### 참고 지표 (Nice to have)

9. 기능 요청 빈도
10. 이탈 시점 분석

## 💡 핵심 성공 요인

### ✅ DO
- **즉각적인 보상**: 완료하면 바로 크레딧 지급
- **선택권 부여**: 강제하지 않고 선택지 제공
- **진행률 시각화**: 프로그레스 바로 동기부여
- **소셜 증거**: "지금 34명이 참여 중!"

### ❌ DON'T
- 너무 긴 설문 (문항 5개 이하)
- 의무적인 단계 (건너뛰기 허용)
- 복잡한 튜토리얼 (바로 체험)
- 스팸성 알림 (주 2회 이하)
